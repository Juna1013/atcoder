# abc444_b ミス記録

## 発生したミス
- ループ内で `tmp = N` にしてしまい、毎回同じ数の桁和を計算していた。
- 桁和の処理が逆 (`sum += tmp / 10`, `tmp %= 10`) になっていた。
- `cnt` を初期化せずに加算していた（未定義動作）。
- 判定対象を `sum` ではなく `cnt` にしていた。
- 探索範囲を `0..N-1` にしていた（問題は正整数 `1..N`）。

## 正しい方針
- `i = 1..N` を走査する。
- `tmp = i` として、`sum += tmp % 10; tmp /= 10;` で桁和を求める。
- `if (sum == K)` を満たす個数を数える。

## 再発防止チェック
- ループ中の対象変数が `i` になっているか。
- `sum` と `cnt` の役割が混ざっていないか。
- `% 10` と `/ 10` の順序が正しいか。
- 初期化漏れの変数がないか（`-Wall -Wextra` で確認）。
