# ABC451 ミスまとめ

## A問題

### バグ内容

`sizeof(配列名)` と `strlen(配列名)` を混同した。

```c
// NG: 配列の宣言サイズ（常に11）が返る
int len = sizeof(S);

// OK: 実際の文字列の長さが返る
int len = strlen(S);
```

### 原因

- `sizeof` は**型・配列のバイト数**を返すコンパイル時定数
- `strlen` は**ヌル文字 `\0` までの文字数**を返す実行時関数

`char S[11]` と宣言した場合、`sizeof(S)` は常に `11` を返すため、入力に関わらず同じ値になってしまう。

### 教訓

文字列の長さが必要なときは必ず `strlen()` を使う。`sizeof()` は配列の要素数確認など**サイズが固定の場合**にのみ使う。

---

### バグ2：Yes/No の出力が逆

条件分岐の出力を逆に書いていた。

```c
// NG: 違反（倍数）のとき "No" を出力してしまっている
if (len % 5 == 0) {
    printf("No\n");
} else {
    printf("Yes\n");
}

// OK
if (len % 5 == 0) {
    printf("Yes\n");
} else {
    printf("No\n");
}
```

### 原因（バグ2）

バグ1（`sizeof`）のせいで `len` が常に `11` になっており、`11 % 5 != 0` で常に `else` に入っていた。
入力例1（"legal" → 期待値 `Yes`）だけで確認していたため、たまたま通ってしまい Yes/No の逆転に気づかなかった。

### 教訓（バグ2）

- `sizeof` バグが別のバグを隠蔽していた（バグの連鎖）
- 手元テストは**全入力例を確認**してから提出する
- 出力の Yes/No は問題文の条件と照らし合わせて必ず確認する

## B問題

### バグ1：VLA を `= {0}` で初期化しようとした

C99 の可変長配列（VLA）は `= {0}` による初期化ができない。ループで明示的に初期化する必要がある。

```c
// NG: VLA はコンパイルエラーになる
int dep_now[M] = {0};

// OK: ループで初期化
int dep_now[M + 1];
for (int i = 0; i <= M; i++) {
    dep_now[i] = 0;
}
```

### バグ2：1-indexed の配列を 0-indexed のサイズで宣言した

部門番号は `1〜M` だが、配列サイズを `M` にしていたため添字 `M` が範囲外アクセスになっていた。また出力ループが `0〜M-1` になっており、データの格納範囲（`1〜M`）とずれていた。

```c
// NG: サイズM の配列に添字1〜M でアクセス → 添字M が範囲外
int dep_now[M];
dep_now[A]++;  // A は 1〜M

// OK: サイズを M+1 にして出力も 1〜M でループ
int dep_now[M + 1];
dep_now[A]++;

for (int i = 1; i <= M; i++) {
    printf("%d\n", dep_next[i] - dep_now[i]);
}
```

### 教訓（B問題）

- ローカルの VLA は `= {0}` で初期化できない。必ずループで初期化する
- 問題の番号が **1-indexed** のときは配列サイズを `N+1` にして添字をそのまま使うか、`-1` して 0-indexed に変換するかを明示的に決める
