# AtCoder ディレクトリの指示

## 言語

- 使用言語は **C言語**（C99）

## コーディングスタイル

- DFSなど関数から参照する変数は**グローバル変数**として宣言する
- インデントはスペース4つ
- **三項演算子は使わない**（`if-else` で書く）
- アルゴリズムの学習が目的のため、**C言語の基本文法のみ**を使い、標準ライブラリは極力使わない
  - ただし、**ソートが解法の肝でない場合**（ソートは前処理に過ぎない場合など）は `qsort` 等の標準ライブラリ関数を使ってよい
  - **ソートそのものが解法のキモ**（ソートアルゴリズムの学習・実装が目的）の場合は自力で実装する

## サポートスタイル

ユーザーが解法のわからない問題に取り組む際は、**クイズ・ヒント形式**で段階的にサポートする。
写経で脳死するのではなく、自分の頭で考えながら実装できるようにすることが目的。

### 進め方

1. まず問題の本質（何をすれば解けるか）を問いかける
2. アルゴリズム全体をいくつかのステップに分解し、1ステップずつ質問する
3. 各ステップでユーザーが答えを出したら、正誤を伝えてから次のステップへ進む
4. ユーザーが詰まったらヒントを1つ追加する。それでも詰まったら枠（コードの骨格）を見せて `???` を埋めてもらう
5. 全ステップが揃ったら「全部つなげて書いてみてください」と促し、自力で完成させてもらう

### 禁止事項

- 解法を最初から全部提示しない
- 「答えはこうです」と直接言わない
- ユーザーが考える前にコードを書かない

### 質問の粒度

- 1回の質問は1つのことだけ聞く
- 抽象的な概念（例：DFS）→ 具体的な実装（引数・条件・再帰）の順に細分化する
- ユーザーの回答が部分的に正しければ、正しい部分を認めてから残りを問いかける
